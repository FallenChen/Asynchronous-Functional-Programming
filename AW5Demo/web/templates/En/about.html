About

The AgileWiki project is the result of my efforts to develop a programming practice based on Norman Kashdan's Rolonics Theory, and has progressed significantly over the years. The focus of this site then is on AgileWiki 5, which benefits both from a larger group effort and a corporate sponsor, Tasaheel Information & Electronic Co.

New in AgileWiki 5

AgileWiki 5 is still in early alpha, which means that a lot of things--like wiki pages--have not yet been implemented. But the basic framework is complete, and there are a few things well worth looking at.

A powerful template engine has been introduced. Templates preserve a clean separation of concerns, but AgileWiki 5 templates go a bit further as they provide the front-end developer with the tools needed to create complete applications.

Application logic is implemented with a collection of Groovy scripts, which are weaved together with the AgileWiki 5 templates.

The greatest effort in the development of AgileWiki 5 has been in making sure that everything will scale nicely. Indeed, we expect applications to easily scale to a few hundred servers. A lot of work remains in this area, especially testing and the development of configuration and support tools for managing a server farm. Our approch is to make scaling up a matter of configuration, independent of the application. AgileWiki 5 then may well be the first of a kind, a fully scalable application platform.

Bill La Forge, Project Lead
