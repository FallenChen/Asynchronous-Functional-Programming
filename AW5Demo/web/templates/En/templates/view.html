The aw:view element is used to include a view template after validating access.

Attributes

One attribute is required, path, which provides the absolute pathname of the view template.

Parameters

One parameter is required, rolonUuid, which provides the UUID of the Rolon that has been loaded into the context map.

Example

Here's a sample:

<aw:query cmd="singletonUuid" role="users"> <aw:query cmd="rolon"> <aw:view path="/views/rolon/frog.xml" /> </aw:query> </aw:query>
