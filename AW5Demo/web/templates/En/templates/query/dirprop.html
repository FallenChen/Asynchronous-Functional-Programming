The dirProp query command adds the properties from a selected dir.prop file to the context map. If no dir.prop is found, then no updates are made to the context map.

Attributes

The cmd attribute on the aw:query element is required to invoke this command: 

cmd="dirProp"

Parameters

This command uses two optional parameters from the context map:

The path parameter
provides an absolute or relative pathname to the dir.prop file to be processed. If not present, a default value of "." is used.
The dirPrefix parameter
provides a string which is pre-pended to the names of the properties in the dir.prop file. If not present, a default value of "dir" is used.
