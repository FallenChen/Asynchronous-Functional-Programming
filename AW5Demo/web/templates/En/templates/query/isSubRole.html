The isSubRole query command determines if one role is a sub-role of another role.

Attributes

The cmd attribute on the aw:query element is required to invoke this command: 

cmd="isSubRole"

Parameters

This command uses two parameters:

The role1 parameter
provides the name of a role. This is a required parameter and must be present in the context map.
The role2 parameter
provides the name of a role. This is a required parameter and must be present in the context map.

Results

If role1 is a sub-role of role2, then the constant isSubRole is set to "true".

Example

Here is a sample usage of isSubRole: 

    <p>
    <aw:query cmd="isSubRole" role1="user" role2="qualifier">
      user is
      <aw:nis name="isSubRole">
        not
      </aw:nis>
      a sub-role of qualifier.
    </aw:query>
    </p><p>
    <aw:query cmd="isSubRole" role1="qualifier" role2="user">
      qualifier is 
      <aw:nis name="isSubRole">
        not
      </aw:nis>
      a sub-role of user.
    </aw:query>
    </p>

And here's the result:

user is not a sub-role of qualifier.

qualifier is not a sub-role of user.
