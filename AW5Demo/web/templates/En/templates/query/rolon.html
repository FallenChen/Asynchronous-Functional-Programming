The rolon query command pulls information from the selected rolon and adds it to the context map. If this information was already present in the context map, or if the rolon does not exist, then no updates are made to the context map.

Attributes

The cmd attribute on the aw:query element is required to invoke this command: 

cmd="rolon"

Parameters

The rolonUuid parameter provides the uuid of the rolon and must be present in the context map.

Example

Here's the code to access the home rolon: 

    <aw:query cmd="singletonUuid" role="users">
      <aw:query cmd="rolon">
        <p>
          <b>Ark Rolon:</b>
        </p>
        <aw:query cmd="versionId" aw:uuid="rolonUuid">
          <aw:include path="dump.xml" aw:setcon.dump.prefix="versionId" />
        </aw:query>
      </aw:query>
    </aw:query>

Note in the above, we use singletonUuid to set the rolonUuid, based on the role. We can do this because the home role is a singleton, i.e. there is only a single rolon for that type of role.
