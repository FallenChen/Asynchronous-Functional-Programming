The versionId query command creates a version id from the current timestamp and the uuid parameter. This is often needed as the content of a Rolon is stored in the context map using the version id as a prefix.

Attributes

The cmd attribute on the aw:query element is required to invoke this command: 

cmd="versionId"

Parameters

This command uses one parameter which must be present in the context map:

The uuid parameter
provides the uuid to be used in creating the timestamp.

Constants

This command adds one constant to the context map: versionId.

Example

    <aw:query cmd="singletonUuid" role="home">
      <aw:query cmd="versionId" aw:uuid="rolonUuid">
        ark version id = <aw:contents name="versionId" />
      </aw:query>
    </aw:query>

And here's the result:

ark version id =
