The Url Append query command appends values of name and value parameters to the url. If url variable is not present in context, or has a zero length,it take the value of currentRequest. The url variable is updated with appended url.

Attributes

The cmd attribute on the aw:query element is required to invoke this command: 

cmd="urlAppend"

Parameters

This command uses three parameters, out of which two must be present in the context map and one is optional:

The url variable
provides the url. This is optional - the variable will be created if necessary, but then the scope is limited.
The name parameter
provides the name variable in the context map.
The value parameter
provides the value of name variable.

Example

Here is the code to access url append: 

       <aw:noop newvar.url="xyz">
         <aw:query cmd="urlAppend" name="foo" value="bar" />
         <aw:query cmd="urlAppend" name="secret" value="plugh" />
         url = <aw:contents aw:value="url" />
       </aw:noop>
