The aw:is and aw:nis Elements

An application will often require the conditional generation of HTML elements. This is handled by testing for the presense of abscence of a name/value pair in the context map through the use of the aw:is and aw:nis elements.

The aw:is and aw:nis elements take either a name or a value attribute. If the given name is present in the context map and the assigned value has a length > 0, then aw:is processes its subordinate elements; otherwise it does not. The reverse applies to the aw:nis element.

If instead of a name attribute, a value attribute is used and the value assigned to this attribute has a length > 0, then aw:is processes its subordinate elements; otherwise it does not. Again, the reverse applies to the aw:nis element. 

<p setcon.x="1">
	<aw:is name="x">
		Frankly
    </aw:is>
    <aw:is name="y">
        Honestly,
    </aw:is>
    <aw:nis name="x">
        quite unlikely!
    </aw:nis>
    <aw:nis name="y">
        not!
    </aw:nis>
</p>

The above template code generates the HTML below: 

<p>
	Frankly
    	not!
</p>
