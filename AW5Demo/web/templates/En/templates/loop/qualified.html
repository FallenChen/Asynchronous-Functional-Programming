The qualified sequence command supports looping through the qualified Rolons of a qualifier Rolon and is invoked using the aw:loop element. The series of keys generated by this command are the UUID's of the qualified Rolons.

Attributes

The cmd attribute on the aw:loop element is required to invoke this command: 

cmd="qualified"

Parameters

The qualified sequence command requires one parameter, whose name may vary depending on the value of the loopPrefix parameter used by the aw:loop element:

rolonUuid
The UUID of the qualifier Rolon.

Loop Constants

The uuid constant is set to the UUID of each child Rolon. (The name of this parameter will vary, depending on the value of the loopPrefix parameter.)

Example

In this example we list all the Rolons which are qualified by the access rolon: 

    <a name="qualified" />
    <aw:noop newvar.resetLink="" newvar.previousLink="" newvar.nextLink="">
      <table border="1">
        <tr>
          <th></th>
          <th>
            Role
          </th>
          <th>
            UUID
          </th>
        </tr>
        <aw:query cmd="singletonUuid" role="access">
          <aw:query cmd="rolon">
            <aw:loop cmd="qualified" limit="4" anchor="qualified">
              <tr aw:class="stripeClass">
                <td align="right">
                  <aw:contents name="index" />
                </td><td>
                  <aw:query cmd="versionId">
                    <aw:contents aw.versionId:value="role" />
                  </aw:query>
                </td><td>
                  <aw:contents name="uuid" />
                </td>
              </tr>
            </aw:loop>
          </aw:query>
        </aw:query>
      </table>
      <aw:is name="previousLink">
        <a aw:href="resetLink">Reset</a>
        <a aw:href="previousLink">Previous</a>
      </aw:is>
      <aw:is name="nextLink">
        <a aw:href="nextLink">Next</a>
      </aw:is>
    </aw:noop>

Here is the result of the above:

Role UUID
  • Next
  • Previous
