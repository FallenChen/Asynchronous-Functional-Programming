The context sequence command supports looping through the context map and is invoked using the aw:loop element. The series of values generated by this command are the names in the context map.

Attributes

The cmd attribute on the aw:loop element is required to invoke this command: 

cmd="context"

Example

Here's the dump.xml template, which displays the contents of the context map: 

<aw:noop newvar.dump.resetLink="" newvar.dump.previousLink="" newvar.dump.nextLink="">
  <table border="1">
    <tr>
      <th></th>
      <th>
        Attribute
     </th>
      <th>
        Type
      </th>
      <th>
        Value
     </th>
    </tr>
    <aw:loop cmd="context" loopPrefix="dump">
      <tr aw:class="dump.stripeClass">
        <td align="right">
          <aw:contents name="dump.index" />
        </td><td>
          <aw:is name="dump.prefix">
            <aw:contents aw-rmpre:value="dump.key" />
          </aw:is>
          <aw:nis name="dump.prefix">
            <aw:contents name="dump.key" />
          </aw:nis>
        </td><td>
          <aw:query cmd="contextType" aw:name="dump.key">
            <aw:contents name="type" />
          </aw:query>
        </td><td>
          <aw:contents aw:name="dump.key" />
        </td>
      </tr>
    </aw:loop>
  </table>
  <aw:is name="dump.previousLink">
    <a aw:href="dump.resetLink">Reset</a>
    <a aw:href="dump.previousLink">Previous</a>
  </aw:is>
  <aw:is name="dump.nextLink">
    <a aw:href="dump.nextLink">Next</a>
  </aw:is>
</aw:noop>
