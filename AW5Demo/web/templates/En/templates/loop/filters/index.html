Filters are used when processing the aw:loop element to prevent further processing of selected keys. They are also used by the contextFilterSequence. (See the extended context.)

Filters work by assigning a non-empty string to the ignore variable for items in the sequence to be ignored. Filters can also add additional values to the context map, as is the case in the children sequence command.

An advance use of filters is to extend the key. This feature is only available when the contextFilterSequnce is invoked with a delimiter. In this case, the extension to the key is assigned to the extension variable.

Note that both the ignore and the extension variable names will be prepended by the value of loopPrefix, when present.
