The postfix filter command is used to prevent further processing of keys which do not end with a given postfix.

Attributes

Filters use 2 attributes specified on the aw:loop element:

The filter attribute on the aw:loop element
is required to invoke this command:

filter="postfix"

The loopPrefix
when specified as an attribute on the aw:loop element, is pre-pended to the parameters used by the postfix filter command.

Properties

The postfix property is used by this command to determine which keys are not to be processed further. Note that the name of this property is modified when a loopPrefix is specified.

Example

The context.xml template uses this command to select properties which end in ".html": 

<p aw:setcon.path="pagePathname">
  <ul>
    <aw:query cmd="dirProp">
      <aw:loop cmd="context" prefix="dir" filter="postfix" postfix="html">
        <li>
          <aw:is aw.key:value="link">
            <aw:a aw.key:href="link"><aw:contents aw:name="key" /></aw:a>
          </aw:is>
          <aw:nis aw.key:value="link" aw:newvar.tmp="pageDirectory">
            <aw:query cmd="append" name="tmp" value="/" />
            <aw:query cmd="append" name="tmp" aw-rmpre:setcon.value="key" />
            <aw:a aw:href="tmp"><aw:contents aw:name="key" /></aw:a>
          </aw:nis>
        </li>
      </aw:loop>
    </aw:query>
  </ul>
</p>
