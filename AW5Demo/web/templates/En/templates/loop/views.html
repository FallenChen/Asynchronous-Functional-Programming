The views sequence command supports looping through the accessible views of a Rolon and is invoked using the aw:loop element. The series of keys generated by this command are the view names.

The views of the role of the selected Rolon are processed first, and then the super-roles, depth first and in configuration order. When a view is found with the same name as a previouly encountered and accessible view, the new view is ignored. By this means a view can be overridden.

All view candidates are first processed, as above, before any looping occurs. Sequencing throough the accessible views is then done in alphabetical order by view name.

Attributes

The cmd attribute on the aw:loop element is required to invoke this command:

cmd="views"

Parameters

The parents sequence command requires a single parameter, rolonUuid, whose value is the UUID of the selected Rolon that has been loaded into the context map. (The name of this parameter will vary, depending on the value of the loopPrefix parameter.)

Loop Constants

The path constant is set to the absolute path of each view, in turn. (The name of this parameter will vary, depending on the value of the loopPrefix parameter.)

The desc constant is set to the description of each view, in turn. (The name of this parameter will vary, depending on the value of the loopPrefix parameter.)

The loader constant is set to the path for the loader of each view, in turn. The loader is the top-level html page which loads the java script libraries for that type of view and then initiates the loading of the remainder of the page. (The name of this parameter will vary, depending on the value of the loopPrefix parameter.)

View Browser

The View Browser is implemented using the views sequence command.
