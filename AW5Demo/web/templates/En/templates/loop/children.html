The children sequence command supports looping through the children of a Rolon and is invoked using the aw:loop element. The series of keys generated by this command are the names of the child Rolons.

Attributes

The cmd attribute on the aw:loop element is required to invoke this command: 

cmd="children"

Parameters

The children sequence command requires one parameter, whose names may vary depending on the value of the loopPrefix parameter used by the aw:loop element:

rolonUuid
The UUID of the parent Rolon. The Rolon must have already been read into the context map.

Loop Constants

The uuid constant is set to the UUID of each child Rolon. (The name of this parameter will vary, depending on the value of the loopPrefix parameter.)

Example

In this example we list all the users, which are the children of the home rolon: 

<p newvar.resetLink="" newvar.previousLink="" newvar.nextLink="">
  <table border="1">
    <tr>
      <th></th>
      <th>
        Name
      </th>
      <th>
        Role
      </th>
      <th>
        UUID
      </th>
    </tr>
    <aw:query cmd="singletonUuid" role="users">
      <aw:query cmd="rolon">
        <aw:loop cmd="children" limit="3">
          <tr aw:class="stripeClass">
            <td align="right">
              <aw:contents name="index" />
            </td><td>
              <aw:contents name="key" />
            </td><td>
              <aw:query cmd="versionId">
                <aw:contents aw.versionId:value="role" />
              </aw:query>
            </td><td>
              <aw:contents name="uuid" />
            </td>
          </tr>
        </aw:loop>
      </aw:query>
    </aw:query>
  </table>
  <aw:is name="previousLink">
    <a aw:href="resetLink">Reset</a>
    <a aw:href="previousLink">Previous</a>
  </aw:is>
    <aw:is name="nextLink">
    <a aw:href="nextLink">Next</a>
  </aw:is>
</p>
