UpdateRequests is simply a list of UpdateRequest objects. This list is accessible via the Extended Context. Updates to the database are defined by a template, which composes one or more update requests and adds them to the updateRequests list. This list is subsequently applied against one or more databases on completion of template processing.

Each update request applies to a single database and is processed atomically as a single transaction.

At this time, only a single update request can be added to the updateRequests list. Before we can process multiple requests, a locking system needs to be put in place, locking the appropriate Rolons in UUID order prior to processing any update requests. Request processing will also need to be made persistent and resumed on restart. Otherwise there is a significant risk of partial failure.
