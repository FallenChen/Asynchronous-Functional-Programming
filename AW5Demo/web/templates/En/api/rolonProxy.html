A Rolon proxy is a limited representation of either a Rolon which is present in the context map or a Rolon which is to be created.

A Rolon proxy can be created from a Rolon already loaded into the context map using the rolonProxy method of the extended context object. A create child update request object can also be used as a Rolon proxy, as it is a subclass.

String

getRoleName()

Returns the name of the Rolon's role.

String

getUuidParameterName()

Returns the name of the UUID parameter used as a key in the update parameters map.

String

getLastCommand()

Returns the command id of the last command applied to the Rolon.
