A noop update request is an update request which does not update the database. This request is helpful for when a template is composing updates and finds that either the user input is in error or that the database is in an inappropriate state. The template can then post an error via the update parameters map and the presense of the noop update request in the update requests list will cause the requested template, or an alternative, to be processed.

A noop update request is created by calling noopUpdateRequest factory method on the extended context object, after which it needs to be added to the update requests list.

Factory Arguments

The factory method in the extended context requires two arguments, which are common to all update requests:

String requestName
The name of the update request. Each update request must have a unique request name. The request name is used as a prefix for any results added to the update parameters map when the request is handled. (The noop update request, however, adds nothing to the update parameters list - it has no results.)
Context context
the context map.
