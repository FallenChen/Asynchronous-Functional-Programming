Every Rolon has a role, which defines its behavior. For example, only Rolons with a role of qualifier, or a role which subclasses the qualifer role, can be used as a qualifier.

A role may extend any number of other roles, i.e. multiple inheritence is supported.

A role can be accessed, given the name of a role, via the Extended Context.

String

roleName()

Returns the name of the role.

boolean

isA(String roleName)

Returns true when the named role is a super-role.

String

parentRoleName()

Returns the required role of the parent Rolon, or null if no parent is allowed.

String

childRoleName()

Returns the required role of the child Rolon, or null if no child is allowed.

String

qualifiedRoleName()

Returns the required role of the qualified Rolon, or null if the role is not a qualifier.

String

property(String propertyName)

Returns the role property, or null if there is no such property.
