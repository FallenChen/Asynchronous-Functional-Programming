Web Application Development frameworks tends to follow a path of evolution rather than revolution, well AgileWiki is intended to be revolutionary and is breathtakingly successful in its aim.

AgileWiki will make you look at that science of building Web Applications in a different way. Built upon the principles of Rolonic Theory, AgileWiki takes Web Application Development to the next level. Scaling is built in and applications built upon the AgileWiki Framework are immediately suited to life in The Cloud.

AgileWiki is written in Scala 2.8 and uses an embedded Jetty 7 web server, two new but crucially stable technologies. Comet based push communication comes as standard. AgileWiki is of course a NoSQL distributed database based solution, (Did you even have to ask?), this means that it easily spans petabytes of data over hundreds of servers.

Asynchronousity is at the heart of the AgileWiki framework. Asynchronous processes are highly scalable and are key contributors to application robustness.

Deployment is a breeze, the only dependency is the Java Virtual Machine. We do the rest for you. Developers can craft applications combining AgileWiki's powerful new templating system and Groovy scripts. No Scala knowledge is required. In fact we positively encourage you to rethink your approach to Web Application Development.

Playing nicely with HTML5 and JavaScript libraries means that Front End Developers can easily integrate increasingly important client-side functionality and Web Designers can bring sites alive without worrying about any underlying complexity.

Did we mention that it scales?
